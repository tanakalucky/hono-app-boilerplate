# AIワークスペース

このディレクトリは、AIエージェントが生成する一時的なドキュメントを保管します。

## ディレクトリ構造

```
.ai-workspace/
├── README.md                    # このファイル
└── {branch-name}/               # ブランチごとのワークスペース
    ├── 01-requirements.md       # 要件定義書
    ├── 02-design.md             # 設計書（後続工程で作成）
    ├── 03-tasks.md              # タスクリスト（後続工程で作成）
    └── assets/                  # 図表など
```

## ファイル命名規則

- **番号プレフィックス**: 作成順序を示す（01, 02, 03...）
- **英語ファイル名**: 日本語ファイル名は避ける（英語のケバブケース）
- **Markdownフォーマット**: すべて `.md` 形式

## 使い方

### 要件定義エージェント

要件定義エージェント (`requirements-analyzer`) を呼び出すと、対話を通じて要件を整理し、`{branch-name}/01-requirements.md` を生成します。

```bash
# 使用例
/task @requirements-analyzer "ユーザー認証機能の要件を定義したい"
```

### 設計エージェント（今後実装予定）

設計エージェント (`design-architect`) を呼び出すと、要件定義書を元に設計書を生成します。

### タスク分解エージェント（今後実装予定）

タスク分解エージェント (`task-breakdown`) を呼び出すと、設計書を元に実装タスクを分解します。

## 注意事項

- このディレクトリは `.gitignore` に追加されており、Gitで管理されません
- ブランチごとに独立したワークスペースを持ちます
- ドキュメントは一時的なものであり、最終的には正式なドキュメントに統合されます
