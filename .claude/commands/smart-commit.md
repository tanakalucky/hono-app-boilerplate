git diffを分析して、変更を適切な単位に分割し、日本語のConventional Commitメッセージを生成してコミットしてください。

- 作業ルール
  - コミット規約（.claude/skills/commit-convention.md）に厳密に従ってください
  - 変更内容をセマンティックに分析し、論理的な単位で分割してください
  - 各コミットは単一の責務を持つようにしてください
  - コミット分割の優先順位: 依存関係 > 設定 > 機能 > バグ修正 > リファクタ > ドキュメント > スタイル > テスト
  - コミットメッセージは日本語で明確に記述してください（type(scope): メッセージ形式）
  - コミット実行前に必ずユーザーに確認を求めてください
  - 各コミット提案には以下を含めてください：
    - コミットメッセージ（type(scope): メッセージ形式）
    - 対象ファイル一覧
    - 変更内容の要約
  - ユーザーが承認したコミットのみを実行してください
  - すべてのコミット完了後、git logで結果を表示してください
